# 🌌 宇宙探针：基于测光数据的恒星-星系-类星体分类挑战

## 题目背景

在现代天文学中，大视场巡天项目每晚产生海量的天体图像与测光数据。以斯隆数字化巡天（Sloan Digital Sky Survey, SDSS）为代表的第五代光谱巡天（SDSS-V），已累计发布超过 $4.6\times10^6$ 条有光谱确认的天体记录，为研究宇宙大尺度结构、星系演化以及超大质量黑洞的活动提供了无可替代的数据宝藏。

然而，面对数十亿计的天体候选体，光谱观测的昂贵与耗时使得只有极少数天体能够获得确切的类型标签。AI for Science 的浪潮为这一困境带来了转机：通过海量有光谱标签的天体测光数据训练分类模型，天文学家能够高效地从巡天图像中筛选出高价值的观测目标（尤其是罕见的类星体），极大提升后续大型望远镜的观测效率。

本题的灵感来源于 ArXiv 最新天文学研究：“基于机器学习与符号回归的 SDSS 天体自动分类”（参考论文见后）。你的任务是扮演 **“数字天文学家”** ，利用天体在五个光学波段的测光特征（星等），构建模型将其精准划分为三类：**恒星（STAR）**、**星系（GALAXY）** 和 **类星体（QSO）**。

类星体是遥远的活动星系核，中心通常藏匿着快速增长的超大质量黑洞，是研究早期宇宙的关键探针。然而，它们在数据集中占比极低（约 $18.8\%$），且极易与恒星或致密星系混淆。**本题仅以 $F_2$ 分数作为评价指标**，正是为了强调召回率（Recall）的重要性——**漏掉一个类星体的科学代价远高于误判几个普通天体**。

## 题目描述

你需要完成以下任务：

1. 理解并处理 SDSS 测光星表数据；
2. 设计并训练一个分类模型；
3. 输入：天体的五个波段（$u, g, r, i, z$）模型星等（`modelMag`）；
4. 输出：该天体所属类别（`GALAXY`, `STAR`, `QSO`）；
5. **最终排名仅依据模型在 B 榜测试集上预测结果与 Ground Truth 的 $F_2$ 分数**。

数据集共计 $90000$ 条记录，每条记录均来自 SDSS 光谱确认的天体，具有极高的标签可信度。数据已划分为训练集、A 榜测试集与 B 榜测试集，具体划分见下文。

本题数据基于 **SDSS Data Release 17** 的公开星表，使用 `PhotoPrimary` 视图与 `SpecObj` 光谱表关联，筛选出具有光谱分类标签 `GALAXY`、`STAR` 或 `QSO` 的天体。原始数据总量为 $4,618,598$ 条，按比例随机分割为以下 CSV 文件。

### 评估指标

本题采用 **$F_2$ 分数（F2 Score）** 作为唯一评价指标。$F_2$ 分数是精确率（Precision）和召回率（Recall）的加权调和平均，其中召回率的权重是精确率的 $4$ 倍，定义为：

$$
F_2 = (1+2^2) \cdot \frac{\text{Precision} \cdot \text{Recall}}{(2^2 \cdot \text{Precision}) + \text{Recall}}
= 5 \cdot \frac{\text{Precision} \cdot \text{Recall}}{4 \cdot \text{Precision} + \text{Recall}}
$$

$F_2$ 分数赋予召回率更高的权重，意味着模型 **“宁可错判，不可漏判”** ，尤其需要尽可能多地找出稀有的 `QSO`（类星体）样本。因为在实际天文巡天中，漏掉一个高红移类星体可能导致错失一次重要的早期宇宙发现机会。

**计算方式**：对测试集所有样本计算混淆矩阵，然后对三个类别（`GALAXY`, `STAR`, `QSO`）分别计算精确率与召回率，进而计算各类别的 $F_2$ 分数，最后采用 **加权平均（weighted average）** 得到最终的 $F_2$ 分数（权重为各类别在测试集中的真实样本数比例）。

### 提交格式

选手需对 `A.csv` 和 `B.csv` 分别生成预测结果文件，命名为 `A_predict.csv` 和 `B_predict.csv`。

每个预测文件必须包含以下两列：

| 列名    | 数据类型 | 描述                                                 |
| :------ | :------- | :--------------------------------------------------- |
| `objid` | integer  | 与原始测试文件中的 `objid` 完全一致                  |
| `type`  | string   | 预测的天体类别，取值必须为 `GALAXY`、`STAR` 或 `QSO` |

提交文件示例：

```csv
objid,type
1237645879551000764,GALAXY
1237645879551066262,STAR
1237645879562862699,QSO
```

> **注意**：`objid` 的顺序需与测试文件保持一致，行数必须严格相等，否则该提交无效。

### 约束条件

1. 不得使用任何在天文或通用图像数据集上预训练的模型权重（如 ImageNet 预训练的 CNN，或已在天文数据上预训练的网络）。模型必须从随机初始化开始训练；
2. 仅可使用 NOAI 考纲所允许的第三方库（如 `numpy`, `pandas`, `scikit-learn`, `torch`, `matplotlib` 等）；
3. 不得调用外部大模型 API（如 GPT、Claude 等）进行预测或特征生成；
4. 合理控制模型复杂度与训练时间（建议参数量不超过 $10^6$，单次训练时间不超过 $10$ 分钟）；
5. 最终需提交 `A_predict.csv` 与 `B_predict.csv` 两个文件。
